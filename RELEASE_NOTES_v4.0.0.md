# Release Notes - v4.0.0 (2026-07-19)

## Overview

v4.0.0 is a major structural refactor release. The project moves from 6 independent codebases to a unified source repository with shared code layer + DB adapters. Includes 10 new SDD and search features, OpenSpec spec-driven development, and automated build system.

## Structural Changes

- **Unified source repository** (`/root/ai-agent-infra/`): shared/ (70% shared code) + adapters/{oracle,pg,yashandb}/ (30% DB-specific)
- **build.py**: One-command build generates 6 release zips from single source
- **VERSION single source**: One file, injected into all 6 editions at build time
- **Parameterized tests**: conftest.py with DB-type fixtures
- **OpenSpec Stores**: Shared specs across all 6 editions

## New Features (10 items)

### SDD Enhancement
1. **MCP dynamic tool loading**: Tools loaded from TOOL_REGISTRY WHERE MCP_EXPOSED='Y'
2. **Spec version management**: Version tracking, diff comparison, change history (SPEC_VERSIONS table)
3. **Harness template inheritance**: extends/includes with merged fields (PARENT_TEMPLATE_ID)
4. **DAG visualization + history**: Mermaid flowchart generation + execution log (DAG_EXECUTION_LOG table)
5. **Approval SLA + notifications**: Timeout escalation, auto-escalate (SLA_DEADLINE, SLA_ESCALATE_TO)
6. **Event dead letter queue**: Failed events retry + replay (EVENT_DEAD_LETTER table)

### Search Enhancement
7. **Search result explanation**: vector_score + text_score + graph_path per result (SEARCH_EXPLANATIONS view)
8. **Graph algorithms**: PageRank, shortest path, community detection (Python-side, cross-DB)
9. **Batch + incremental embedding**: Stale detection, batch processing (EMBEDDING_VERSION column)
10. **Agent auto-recovery**: Health check + alert rules + auto-restart (ALERT_RULES table)

## Bug Fixes

- Session cookies now port-specific (session_id_{port}) to prevent cross-edition conflicts
- Auto-logout timer matches session_timeout (3600s, not hardcoded 300s)
- SameSite=Lax on all cookies for tab-switch persistence
- Oracle-specific SQL removed from shared code (DUAL, SYSTIMESTAMP, ROWNUM, SYS_GUID, NVL)
- PG compatibility: col_id->collab_id, branch_status->status, FROM DUAL removed
- LLM empty api_key no longer blocks calls
- reasoning_effort:none for reasoning models
- Version injection handles v3.10.2< and v3.10.2" patterns
- build.py generates README.md from template per edition
- Template placeholders {{EDITION_LABEL}}, {{DB_DISPLAY}}, 4.0.0
